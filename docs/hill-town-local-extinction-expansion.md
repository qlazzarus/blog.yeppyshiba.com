# 언덕마을 지방소멸 컨셉 확장 기획

이 문서는 `hill-town-webgame-plan.md`에 추가할 수 있는 컨셉 확장 문서다.

기존 언덕마을 PoC는 Phaser 아이소메트릭 고저차 렌더링과 Web Worker 기반 시뮬레이션 분리를 검증하는
기술 프로젝트였다. 여기에 한국의 지방소멸 문제의식을 얹어, 단순 도시건설 게임이 아니라
“사람이 떠나는 마을을 어떻게 다시 생활 가능한 곳으로 만들 것인가”를 다루는 작은 웹게임으로 확장한다.

---

## 1. 핵심 컨셉 변경

기존 컨셉:

> 언덕 지형 위에 도로, 집, 밭, 상점이 붙으며 지방 읍이 만들어지는 작은 도시건설 PoC

확장 컨셉:

> 인구가 줄어드는 산 아래 작은 읍에서, 제한된 예산과 지형 조건 속에
> 사람들이 떠나지 않고 머물 수 있는 생활권을 다시 설계하는 아이소메트릭 마을 회복 시뮬레이션

핵심 질문:

```text
어떻게 하면 이 마을에 사람이 계속 살 수 있을까?
```

게임의 목표는 대도시처럼 성장하는 것이 아니다.

목표는 다음에 가깝다.

```text
- 폐교를 막는다
- 청년 가구가 들어올 이유를 만든다
- 노인 가구가 고립되지 않게 한다
- 빈집을 방치하지 않는다
- 작은 일자리와 생활 인프라를 연결한다
- 산과 논밭, 구불구불한 도로를 마을의 제약이자 개성으로 활용한다
```

---

## 2. 게임의 정체성

이 게임은 “큰 도시를 키우는 게임”이 아니라 “작은 읍을 버티게 하는 게임”이다.

비교 대상:

```text
SimCity:
- 큰 도시 성장
- 세금, 전기, 교통, 공업, 범죄, 오염

SimTown:
- 작은 마을
- 생활감
- 집, 학교, 가게, 사람

언덕마을:
- 지방 읍
- 인구감소
- 고령화
- 빈집
- 생활권
- 지형 제약
- 작지만 지속 가능한 마을
```

따라서 수치 목표도 “인구 10만 명 달성” 같은 식이 아니다.

대신 다음 수치를 본다.

```text
- 정착 가구 수
- 빈집 수
- 생활권 접근성
- 고령 가구 고립도
- 청년 유입 가능성
- 아이 돌봄/학교 유지 가능성
- 지역 일자리 안정도
- 마을 만족도
```

## 3. 내러티브 배경

플레이어는 사라져가는 작은 읍의 임시 기획자가 된다.

예시 배경:

```text
산 아래에 있는 작은 읍, 하늘군 도래면.

한때는 초등학교, 시장, 정미소, 버스정류장, 작은 병원이 있던 곳이지만
지금은 인구가 줄고 빈집이 늘어나는 중이다.

큰 공장을 유치할 돈도 없고,
고속철도역을 새로 만들 수도 없다.

플레이어가 할 수 있는 일은 작다.

빈집을 고치고,
길을 잇고,
마을회관 근처에 생활시설을 붙이고,
청년이 들어와 살 수 있는 작은 일거리를 만들고,
고령 주민이 고립되지 않게 하는 것.

이 마을은 대도시가 되지 않는다.
하지만 사라지지 않을 수는 있다.
```

## 4. 게임 목표

### 단기 목표

```text
- 마을회관과 도로를 연결한다
- 빈집을 수리한다
- 고령 가구가 마을회관/보건소/정류장에 접근 가능하게 한다
- 작은 상점이 유지될 만큼 주변 집을 모은다
```

### 중기 목표

```text
- 아이가 있는 가구 3세대 이상 유치
- 폐교 위험 낮추기
- 청년 가구가 들어올 수 있는 주거/일자리 조합 만들기
- 빈집 비율 낮추기
- 버스정류장 주변 생활권 만들기
```

### 장기 목표

```text
- 10년 동안 읍 소멸 위험을 낮춘다
- 인구 규모보다 세대 구성 균형을 맞춘다
- 고립 가구를 줄인다
- 작은 경제 순환을 만든다
```

## 5. 핵심 지표

기존의 `money`, `population`만으로는 지방소멸 컨셉이 약하다.  
아래 지표를 단계적으로 추가한다.

### 5.1 인구 지표

```ts
export type PopulationStats = {
    total: number;
    children: number;
    youth: number;
    workingAge: number;
    elderly: number;
    households: number;
};
```

초기에는 세부 인구를 실제 주민 단위로 시뮬레이션하지 않는다.  
집 타일마다 간단한 가구 타입만 둔다.

```ts
export type HouseholdType =
    | 'empty'
    | 'elderly_single'
    | 'elderly_couple'
    | 'young_single'
    | 'young_couple'
    | 'family_with_child'
    | 'return_farmer';
```

### 5.2 마을 지속성 지표

```ts
export type TownVitalityStats = {
    extinctionRisk: number; // 0~100, 높을수록 위험
    settlementScore: number; // 정착 가능성
    accessScore: number; // 생활권 접근성
    jobScore: number; // 일자리 안정성
    careScore: number; // 돌봄/보건 접근성
    vacancyRate: number; // 빈집 비율
};
```

게임 화면에서는 너무 많은 숫자를 보여주지 않는다.  
초기 HUD는 4개만 보여준다.

```text
인구
빈집
생활권
소멸위험
```

## 6. 주요 자원

기존 자원:

```text
돈
인구
월수입
```

확장 자원:

```text
예산
인구
빈집
생활권
정착도
소멸위험
```

### 예산

돈은 도시 성장의 자원이 아니라 “한정된 지방 예산”처럼 다룬다.

```text
예산 사용처:
- 도로 보수
- 빈집 수리
- 마을버스 정류장 설치
- 작은 상점 유치
- 보건지소 설치
- 공동 작업장 설치
- 어린이 돌봄 공간 설치
```

### 정착도

정착도는 청년/가족/귀촌 가구가 들어올 가능성을 나타낸다.

정착도에 영향을 주는 요소:

```text
+ 빈집 수리
+ 도로 접근성
+ 일자리 근접
+ 학교/돌봄 접근성
+ 상점/정류장 접근성
- 고립된 주거지
- 너무 많은 빈집
- 생활시설 부족
- 도로 유지비 과다
```

### 소멸위험

소멸위험은 단순히 인구가 적다고 올라가지 않는다.  
마을의 세대 구성과 생활권 붕괴 정도를 반영한다.

예시:

```text
소멸위험 증가:
- 빈집 증가
- 아이가 있는 가구 감소
- 고령 가구 고립
- 상점 폐업
- 학교 폐교
- 도로 유지 실패

소멸위험 감소:
- 가족 가구 유입
- 청년 가구 유입
- 빈집 재생
- 생활시설 접근성 개선
- 작은 일자리 유지
```

## 9. 가구 타입과 건물 연결

집 하나를 단순히 `house`로만 보지 않고, 그 집에 어떤 가구가 사는지를 둔다.

```ts
export type Building = {
    type: BuildingType;
    level: number;
    household?: HouseholdType;
    condition: number; // 0~100
};
```

예시:

```text
빈집:
- condition이 낮음
- 방치되면 주변 정착도 감소
- 수리하면 청년/귀촌/가족 가구 유입 가능

고령 1인 가구:
- 인구는 적지만 careScore에 민감
- 보건지소/버스정류장/마을회관 접근성이 중요

가족 가구:
- school/care 접근성이 중요
- 근처에 일자리나 상점이 있으면 정착 가능성 상승

귀촌 가구:
- 빈집 수리 + 밭/공동작업장 접근성에 반응
```

## 10. 생활권 접근성

지방소멸 컨셉에서 가장 중요한 로직은 “생활권”이다.

단순히 건물이 가까운 것이 아니라, 도로와 경사를 따라 실제로 접근 가능한지가 중요하다.

### 생활권 대상

```text
마을회관
버스정류장
상점
보건지소
학교/돌봄 공간
공동 작업장
```

### 접근성 계산

Worker에서 BFS 또는 Dijkstra로 계산한다.

초기에는 BFS로 충분하다.

```text
- 도로를 따라 이동
- 높이 차이 1까지 이동 가능
- 높이 차이 2 이상은 연결 불가
- 각 집에서 주요 시설까지 몇 칸 이내인지 계산
```

### 접근성 예시 규칙

```text
집 → 마을회관 8칸 이내: 생활권 +1
집 → 상점 10칸 이내: 생활권 +1
집 → 버스정류장 8칸 이내: 정착도 +1
고령 가구 → 보건지소 10칸 이내: careScore +1
가족 가구 → 학교/돌봄 12칸 이내: settlementScore +1
```

## 11. 빈집 시스템

지방소멸 컨셉을 가장 직관적으로 보여주는 요소는 빈집이다.

### 빈집 발생

```text
- 시간이 지나면 일부 고령 가구가 사라지고 빈집이 된다
- 접근성이 낮은 집은 빈집이 될 확률이 높다
- 주변에 빈집이 많으면 정착도가 떨어진다
```

### 빈집 활용

```text
- 수리해서 귀촌 주택으로 전환
- 청년 창업 공간으로 전환
- 공동 작업장으로 전환
- 철거해서 밭/공터로 전환
```

### 게임적 효과

```text
방치된 빈집:
- 주변 정착도 감소
- 소멸위험 증가

수리된 빈집:
- 청년/귀촌/가족 가구 유입 가능
- 예산 소모
- 주변 정착도 증가
```

## 12. 유입 이벤트

대도시 게임처럼 인구가 자동으로 계속 늘어나지 않는다.  
조건을 만족할 때 특정 가구가 들어오는 방식이 좋다.

### 청년 가구 유입

조건 예시:

```text
- 수리된 빈집 있음
- 버스정류장 접근 가능
- 공동 작업장 또는 상점 접근 가능
- 주변 빈집 비율이 너무 높지 않음
```

### 가족 가구 유입

조건 예시:

```text
- 수리된 빈집 있음
- 학교/돌봄 공간 접근 가능
- 상점 또는 정류장 접근 가능
- 생활권 점수가 일정 이상
```

### 귀촌 가구 유입

조건 예시:

```text
- 수리된 빈집 있음
- 밭 또는 공동 작업장 근처
- 도로 접근 가능
- 마을회관 접근 가능
```

## 13. 마을 이벤트

월별 또는 연도별로 작은 이벤트를 발생시킨다.  
초기에는 텍스트 카드 형태로 충분하다.

### 부정 이벤트

```text
폐교 위기:
- 아이가 있는 가구가 3세대 미만이면 발생
- 일정 기간 유지되면 school이 closed_school이 됨

상점 폐업:
- 주변 집 수가 부족하거나 접근성이 낮으면 발생

버스 감편:
- 버스정류장 주변 이용 가구가 너무 적으면 발생

빈집 증가:
- 고령 가구 고립도가 높으면 발생

도로 유실:
- 높은 언덕 도로 유지비가 부족하면 발생
```

### 긍정 이벤트

```text
귀촌 문의:
- 수리된 빈집과 밭이 있으면 발생

주말 장터:
- 밭과 상점이 연결되면 발생

작은 카페 창업:
- 청년 가구 + 빈집 수리 + 도로 접근성이 있으면 발생

마을 축제:
- 마을회관, 상점, 버스정류장 생활권이 연결되면 발생
```

## 14. 정책 카드 시스템

현실의 지방소멸대응기금처럼 “예산을 어디에 쓸 것인가”를 게임화한다.  
정책 카드라는 이름을 붙이면 UI와 게임 목표가 분명해진다.

### 정책 카드 예시

```text
빈집 정비 사업
- 비용: 30
- 효과: empty_house 1개를 renovated_house로 전환
- 부작용: 즉시 인구 증가는 없음

마을버스 정류장 설치
- 비용: 50
- 효과: 주변 주거지 정착도 상승
- 조건: road 옆

공동 작업장 조성
- 비용: 80
- 효과: 청년/귀촌 가구 유입 가능성 증가
- 조건: renovated_house 또는 empty_house 근처

작은 보건지소
- 비용: 100
- 효과: 고령 가구 careScore 상승
- 조건: 마을회관 또는 버스정류장 근처

어린이 돌봄 공간
- 비용: 100
- 효과: 가족 가구 정착도 상승
- 조건: 가족 가구 2세대 이상 또는 학교 근처
```

정책 카드는 건설 버튼보다 더 좋은 표현일 수 있다.

```text
도로를 깐다
집을 짓는다
```

보다

```text
빈집을 고친다
버스정류장을 만든다
공동 작업장을 연다
```

가 컨셉과 더 잘 맞는다.

## 22. 핵심 문장

> 언덕마을은 도시를 키우는 게임이 아니라, 사라지는 마을을 다시 생활 가능한 곳으로 만드는 게임이다.

> Phaser는 보이는 마을을 그리고, Web Worker는 보이지 않는 생활권과 소멸위험을 계산한다.

> 플레이어의 적은 괴물이나 경쟁 도시가 아니라, 빈집과 고립과 생활권 붕괴다.
